# 实施计划

## 1. 任务摘要

增加客户端内置状态岛、送达动画、识别生命周期事件和 uv 双包构建入口。

## 2. 风险等级

`complex`。跨线程 UI 与异步识别结果存在竞态风险，构建流程同时影响两个发行包。

## 3. 相关现有代码

- `core/client/app.py`
- `core/client/shortcut/task.py`
- `core/client/audio/recorder.py`
- `core/client/output/result_processor.py`
- `core/client/output/text_output.py`
- `config_client.py`
- `build.spec`
- `build-client.spec`
- `zip_release.py`

## 4. 待修改文件

上述生命周期、配置、构建和说明文件。

## 5. 待新增文件

- 状态事件模型与控制器
- 动态岛 Tkinter UI
- 状态机测试
- uv 构建脚本和项目依赖元数据

## 6. 范围外内容

- 不调整模型准确率、热词算法和服务端推理。
- 不实现 macOS/Linux UI。
- 不自动发布 GitHub Release，仅生成可上传的双 ZIP。

## 7. 数据库影响

无。

## 8. API 影响

不改变客户端与服务端 WebSocket 协议；只增加客户端进程内事件。

## 9. 业务流程影响

录音开始、录音结束、识别成功、输出完成和失败会驱动 UI；原文字输出流程保持原顺序。

## 10. 事务、并发与幂等性

- UI 队列只保留最新任务状态。
- 终态动画按任务 ID 去重。
- UI 异常被隔离并记录日志。
- 退出时停止线程，避免后台残留。

## 11. 实施步骤

1. 定义状态类型、事件和线程安全发布器。
2. 实现无边框置顶 Tkinter 状态岛及四种动画。
3. 在快捷键、录音发送、最终结果、输出和异常边界发布事件。
4. 增加配置并接入客户端启动/停止生命周期。
5. 增加单元测试和无 UI 模式测试。
6. 增加 pyproject/uv 构建脚本，复用两个 spec 与 zip_release.py。
7. 更新文档和 PyInstaller 收集规则。

所有运行时组件均在客户端进程内，由 `start_client.exe` 自动启动；不新增用户侧命令。

## 12. 构建与测试计划

- 运行新增单元测试及现有可用测试。
- 运行 `compileall`。
- 校验两份 PyInstaller spec。
- 使用 uv 构建脚本至少完成依赖/命令 dry-run；环境允许时构建双包。
- Windows 上手工验证状态岛四种状态。

## 13. 回滚方案

配置关闭动态岛即可回到原行为；代码回滚集中在独立模块和少量事件调用点。

## 14. 待确认问题

无阻塞问题，默认采用屏幕底部中央、小尺寸、180ms 录音显示延迟。
