# 任务

为 CapsWriter-Offline 增加客户端内置的动态状态岛，并把录音到文字输出的关键阶段变成可观察、可测试的状态管线；使用 uv 自动化构建完整包与纯客户端包。

## 手动复杂度覆盖

none

## 验收标准

1. 短按 CapsLock 不显示，长按录音显示动态波形。
2. 松开后切换到识别状态，成功输出文字后播放送达流光并自动隐藏。
3. 连接或处理失败时显示短暂错误状态，UI 故障不得阻断语音输入。
4. 功能可配置关闭，退出客户端时正确释放 UI 线程。
5. 完整包和纯客户端包均包含新功能，并提供 uv 一键构建入口。
6. 核心状态机有自动化测试，客户端 PyInstaller 配置完成验证。
7. 所有运行时新增功能随 `start_client.exe` 自动启动和退出，用户不需要额外命令或进程。

## 约束

- 尽量复用现有 ClientState、ShortcutTask、ResultProcessor 和 Tkinter/PyInstaller 体系。
- 不修改 ASR 模型和服务端识别算法。
- 不依赖外部 PowerShell Indicator。
- 不提供需要用户单独运行的 Indicator、IPC 服务或辅助启动器。
- UI 和状态通知失败时只记日志，不能影响录音、识别和文字输出。
