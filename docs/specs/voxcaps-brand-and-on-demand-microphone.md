# VoxCaps 品牌图标与按需麦克风规格

## Problem Statement

VoxCaps 虽然已经完成项目更名，但 Windows 应用窗口、托盘和可执行文件仍显示 CapsWriter 图标，用户无法从系统界面清楚区分派生版本。

客户端当前在启动麦克风模式后持续保持音频输入流，导致 Windows 长期显示麦克风正在使用。即使未进行语音输入，设备仍处于打开状态，容易引发隐私疑虑、占用冲突和安全软件告警。直接改成按键后才打开麦克风又可能产生设备初始化延迟，造成开头语音丢失。

现有桌面反馈还会在紫色识别态后显示约一段时间的绿色送达动画，显得冗余；默认快捷键也缺少右 Alt，不能满足用户以左右手灵活触发语音输入的需求。

## Solution

为 VoxCaps 建立独立、可编辑且适合 Windows 多尺寸显示的品牌图标，并将其统一应用到窗口、托盘和全部 Windows 可执行文件。

把麦克风从“客户端运行期间常驻打开”改为“按需打开”：托盘就绪后执行一次短暂预热并立即释放；按下延迟触发键时异步打开麦克风，将触发阈值期间的音频保存在内存候选缓冲中；确认长按后把候选音频并入正式录音，短按则丢弃并恢复按键默认行为。释放语音触发键、取消候选、发生错误或退出客户端时，都必须及时关闭麦克风。

CapsLock 与右 Alt 统一使用 250 毫秒阈值。右 Alt 在候选期间与其他键组成 AltGr/系统组合时，取消语音候选并保留组合键语义。连续口述允许上一段识别期间开始下一段录音，但最终结果必须按录音开始顺序上屏。

灵动岛增加琥珀色准备态，用于表示长按已确认但麦克风尚未就绪；成功打开后切换为青蓝录音态。识别结果成功送达后立即隐藏，不再显示绿色完成动画。麦克风不可用时只显示通用托盘权限提示，不显示错误灵动岛。

## User Stories

1. 作为 VoxCaps 用户，我希望应用使用独立的 VoxCaps 图标，以便与原版 CapsWriter 清楚区分。
2. 作为 Windows 用户，我希望任务栏和文件资源管理器中的可执行文件显示清晰图标，以便快速识别程序。
3. 作为高 DPI 屏幕用户，我希望图标在不同缩放比例下保持清晰，以便获得一致的系统体验。
4. 作为项目维护者，我希望保留可编辑的矢量图标源文件，以便后续调整品牌而不损失质量。
5. 作为隐私敏感用户，我希望空闲时麦克风保持关闭，以免 Windows 长期显示正在使用麦克风。
6. 作为语音输入用户，我希望客户端启动后自动验证一次麦克风，以便尽早发现权限或安全策略问题。
7. 作为日常用户，我希望预热完成后立即释放麦克风，以免预热变成常驻占用。
8. 作为语音输入用户，我希望按下语音键时立即准备采集，以免第一个字因设备启动延迟而丢失。
9. 作为 CapsLock 用户，我希望短按仍正常切换大小写，以免语音功能破坏原有键盘习惯。
10. 作为右 Alt 用户，我希望短按仍保留系统默认功能，以免语音功能独占该按键。
11. 作为使用 AltGr 键盘布局的用户，我希望右 Alt 与其他按键的组合保持正常，以便继续输入特殊字符。
12. 作为语音输入用户，我希望单独长按 CapsLock 或右 Alt 超过250毫秒后开始录音，以便用任一只手触发。
13. 作为语音输入用户，我希望阈值期间说出的开头语音被保留，以免必须等待动画出现后才开口。
14. 作为短按用户，我希望候选音频只存在于内存且立即丢弃，以免短按内容被保存或发送。
15. 作为 Windows 用户，我接受短按时麦克风指示短暂闪现，但希望释放后立即消失，以换取长按不漏字。
16. 作为语音输入用户，我希望麦克风真正就绪后才显示正常录音波形，以免界面错误暗示正在录音。
17. 作为设备启动较慢的用户，我希望看到琥珀色准备动画，以便知道程序正在等待麦克风。
18. 作为权限受限用户，我希望麦克风超时后收到清晰的托盘通知，以便检查系统权限或安全软件设置。
19. 作为不同安全软件的用户，我希望错误提示保持通用，不点名任何特定安全产品。
20. 作为桌面用户，我希望麦克风错误不显示红色叹号灵动岛，以免出现突兀的桌面动画。
21. 作为语音输入用户，我希望麦克风失败后下一次长按自动重试，以免必须重启客户端。
22. 作为 CapsLock 用户，我希望已确认的长按语音失败时不要再切换大小写，以免出现意外按键副作用。
23. 作为快速操作用户，我希望同时按下两个语音键时只有第一个键拥有录音会话，以免录音被第二个键意外中断。
24. 作为连续口述用户，我希望上一段识别时仍可开始下一段录音，以免被识别延迟打断思路。
25. 作为连续口述用户，我希望多段结果严格按照录音顺序上屏，以免句子次序颠倒。
26. 作为桌面用户，我希望成功上屏后灵动岛立即隐藏，以免绿色完成动画停留过久。
27. 作为退出程序的用户，我希望所有预热、候选和正式录音资源都被释放，以免退出后仍占用设备。
28. 作为发行包用户，我希望完整包和纯客户端包都包含相同的品牌图标和麦克风行为，以免不同安装方式表现不一致。

## Implementation Decisions

- VoxCaps 标志采用深色圆角方形底和青蓝渐变声波，声波整体形成简化的字母 V；不依赖完整文字或细小麦克风图案。
- 保留可编辑的 SVG 源文件，并生成包含 16、24、32、48、64、128 和 256 像素图层的 Windows ICO。
- 同一品牌资源用于客户端窗口、托盘、客户端可执行文件和服务端可执行文件。
- 引入单一的语音会话协调边界，统一管理预热、候选录音、正式录音、会话所有权、释放和失败恢复，避免快捷键、音频和界面各自维护冲突状态。
- 托盘就绪约1秒后后台预热一次麦克风，最长等待1.5秒；成功或失败后都立即释放，失败不阻止客户端运行。
- 客户端空闲时不保留活动的麦克风输入流。
- CapsLock 和右 Alt 都是延迟触发键，统一使用250毫秒阈值。
- 按键按下时立即异步打开麦克风，并将阈值期间的音频写入有界内存候选缓冲。
- 250毫秒内释放属于短按：关闭麦克风、清空候选缓冲，并恢复按键默认语义。
- 超过250毫秒属于语音意图：候选缓冲并入正式录音，之后的音频继续进入当前语音会话。
- 候选录音不得保存到磁盘、进入识别队列、发送到服务端或参与结果处理，除非触发已确认为长按。
- 右 Alt 候选期间出现其他按键时，取消语音候选并保留完整的 AltGr/系统组合键语义。
- 第一个完成长按确认的触发键成为语音会话所有者；其他触发键不能创建、结束或接管当前会话。
- 长按已确认但麦克风尚未打开时显示琥珀色准备态；就绪后切换为青蓝色录音态。
- 麦克风打开超过1.5秒仍未成功时取消当前语音会话、释放资源、隐藏灵动岛并显示托盘通知。
- 麦克风错误提示只建议检查 Windows 麦克风权限或安全软件设置，不提及具体安全产品。
- 麦克风打开失败时不显示错误灵动岛；已经确认的长按也不补发原按键默认功能。
- 释放会话所有者按键时结束录音并关闭麦克风；退出客户端时取消所有未完成的预热、候选和录音工作并关闭设备。
- 识别中允许启动新的语音会话，每个会话使用独立身份，迟到状态不能覆盖当前桌面反馈。
- 识别结果通过顺序送达机制按录音开始顺序输出；后完成的早期结果到达前，后续结果保持等待。
- 成功送达不是独立可见状态，结果上屏后灵动岛立即隐藏，不显示绿色动画。
- 所有新增时序参数应集中配置并提供中文说明，默认值与本规格一致。
- 现有 Client/Server 协议和服务端模型推理流程保持兼容。

## Testing Decisions

- 测试只断言用户可观察行为和公共状态，不依赖线程名称、私有字段或具体音频库调用顺序。
- 主要测试接缝是语音会话协调器，以虚拟时钟、虚拟麦克风、托盘通知、灵动岛和识别传输替身驱动完整状态变化。
- 语音会话测试覆盖：预热成功与失败、预热立即释放、空闲不占用、短按、长按、候选缓冲提交与丢弃、打开超时、退出释放和失败后重试。
- 快捷键适配器测试使用合成的 Windows 键盘事件，覆盖 CapsLock、右 Alt、AltGr 组合、250毫秒边界、快速连按、按键补发和首键会话所有权。
- 连续口述测试模拟识别结果乱序返回，验证最终严格按照录音开始顺序上屏，并验证旧状态不能覆盖当前录音反馈。
- 灵动岛沿用现有状态机测试方式，增加准备态、成功立即隐藏、无绿色送达动画和麦克风错误不显示的行为测试。
- 构建产物测试验证 SVG 可解析、ICO 包含规定尺寸，并验证完整包和纯客户端包的客户端/服务端可执行文件及托盘资源使用 VoxCaps 图标。
- 真实 Windows 验收包括：启动后麦克风指示仅短暂出现、空闲时消失、按下触发键时出现、释放后消失，以及权限被拒绝时客户端保持运行。
- 真实键盘验收包括：CapsLock 短按切换大小写、右 Alt 短按默认行为、AltGr 组合、两种长按录音和交叉按键。
- 最终重新构建完整 Windows 包和纯客户端包，并执行自动化测试、源码编译检查、ZIP 内容检查和实际图标检查。
- 现有灵动岛状态机和几何测试作为本次测试风格的先例继续保留。

## Out of Scope

- 修改局域网 Server 地址选择、本地/局域网自动回退或模型下载策略。
- 修改 Fun-ASR-Nano、其他识别模型或 Client/Server 传输协议。
- 新增完整图形化设置页面或首次运行向导。
- 针对火绒、Windows Defender 或其他单一安全软件调用专用接口。
- Windows 代码签名、安装器、自动更新和 Microsoft Store 发布。
- 重新设计灵动岛的整体尺寸、位置或除本规格状态之外的动画风格。
- 保存短按候选音频或提供候选音频回放功能。

## Further Notes

- 启动预热和短按候选都会短暂打开麦克风，因此 Windows 麦克风指示可能短暂闪现；本规格解决的是空闲期间持续占用，而不是完全消除所有指示。
- 麦克风设备首次授权或被安全策略检查时可能超过正常打开时间，琥珀色准备态和1.5秒超时用于给出诚实反馈。
- `ready-for-agent` 是本规格发布后的唯一新增 GitHub 标签，不需要引入额外项目级工作流配置。
