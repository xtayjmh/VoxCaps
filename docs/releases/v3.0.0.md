## 版本说明

VoxCaps 基于 [HaujetZhao/CapsWriter-Offline](https://github.com/HaujetZhao/CapsWriter-Offline) 二次开发，不是原项目的官方发行版。本版本保留原项目的离线语音识别、客户端/服务端通信、热词和文本处理能力，并重点增强 Windows 桌面输入体验。

## 相对原仓库的主要新增与调整

- 客户端内置灵动岛：以琥珀、青蓝和紫色分别反馈麦克风准备、录音和识别状态，无需单独启动辅助脚本；
- CapsLock 按需麦克风：启动只预热一次，空闲不持续占用，短按保持大小写功能，长按超过 250 毫秒进入语音输入；
- 候选音频缓冲：按键后立即异步打开麦克风，长按确认时保留开头音频，减少设备启动延迟导致的漏字；
- 连续口述顺序送达：上一段识别期间可继续录音，最终结果严格按照录音顺序上屏；
- 独立 VoxCaps 品牌：统一窗口、托盘、客户端 EXE、服务端 EXE、运行时类名、热词示例及模型 Notebook 元数据；
- 双包分发：同时提供客户端+服务端完整包和纯客户端包，并自动校验测试、版本、图标及 ZIP 内容一致性；
- Server 完整包修复：恢复 Fun-ASR-Nano、Qwen3-ASR 和对齐器共用的 llama.cpp Vulkan 运行库，并补齐 `soundfile/libsndfile`；构建时从 llama.cpp 官方发行版下载固定版本、校验 SHA-256 后打包，用户只需把模型放入对应目录；
- 服务端日志降噪：普通 TCP 端口检查未执行 WebSocket 握手时，不再在黑窗中重复打印 `opening handshake failed` 异常栈；
- 文档增强：补充局域网使用、麦克风权限、灵动岛状态和中文语境夹英文单词的模型选择说明。
