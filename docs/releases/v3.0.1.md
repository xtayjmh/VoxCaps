# VoxCaps 3.0.1

这是 3.0.0 的 Windows Server 修复版本。VoxCaps 基于 [HaujetZhao/CapsWriter-Offline](https://github.com/HaujetZhao/CapsWriter-Offline) 二次开发，不是原项目的官方发行版。

## 修复内容

- 修复完整包遗漏 llama.cpp 原生运行库，导致 Fun-ASR-Nano 与 Qwen3-ASR 在模型检查通过后仍立即退出的问题；
- 四个 GGUF 加载器统一使用 `core/server/engines/llama/bin`，避免不同目录的 DLL 版本漂移；
- 构建时自动下载固定版本的 llama.cpp Vulkan 官方资产并校验 SHA-256，完整包用户只需把模型放进对应目录；
- 补齐 Server 所需的 `soundfile` 与 `libsndfile` 运行时依赖；
- 普通 TCP 端口探测不再重复打印 `opening handshake failed` 异常栈，项目自身的端口占用、启动与业务错误仍正常记录。

## 下载

- `VoxCaps-20260717.zip`：Windows 完整包，包含客户端、服务端以及 GGUF/Vulkan 运行库；
- `VoxCaps-Client-20260717.zip`：Windows 纯客户端包，适合连接局域网服务器。

模型文件继续从原项目的 [Models Release](https://github.com/HaujetZhao/CapsWriter-Offline/releases/tag/models) 下载，并解压到完整包的 `models` 对应目录。

## 验证

- 65 项自动化测试全部通过；
- 源码编译、SVG/ICO 与三个 EXE 图标检查通过；
- 完整包和纯客户端包结构、版本及共享文件一致性检查通过；
- 在实际解压目录完成 Qwen3-ASR 模型加载、Vulkan 后端启动和 WebSocket 6016 端口验证；
- 连续普通 TCP 探测后，服务端错误输出中不再出现无效握手 Traceback。
