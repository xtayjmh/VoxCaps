# 动态状态岛与 UV 双包构建

## 使用方式

动态状态岛已经集成进 `start_client`：启动客户端时自动启动，退出客户端时自动释放，不需要运行额外脚本或后台程序。

状态含义：

- 琥珀波形：长按已确认，正在等待麦克风就绪；
- 青蓝波形：正在录音；
- 紫色光点：服务端正在识别，或客户端正在处理结果；
- 成功上屏后立即隐藏，不显示绿色完成动画；
- 麦克风、连接或发送错误只保留通知或日志，不显示红色错误岛。

短按 CapsLock 切换大小写时，状态岛不会出现。只有按住超过 250 毫秒后才进入准备态。

CapsLock 是唯一的键盘语音触发键。250 毫秒内松开时会关闭候选麦克风、丢弃候选音频并正常切换大小写；右 Alt 与 AltGr 不被语音逻辑接管。由于按键按下时会先异步打开麦克风以避免漏字，短按时 Windows 麦克风指示可能短暂闪现。

客户端启动约 1 秒后执行一次麦克风预热，成功或失败后都会立即释放；空闲时不持续占用麦克风。权限或安全策略阻止访问时，托盘只给出通用的麦克风权限提示，客户端保持运行并在下一次长按时重试。

## 配置

在 `config_client.py` 的 `ClientConfig` 中调整：

```python
dynamic_island_enabled = True       # 是否启用
dynamic_island_width = 276          # 宽度（最小 96）
dynamic_island_height = 68          # 高度（最小 26）
dynamic_island_bottom_margin = 18   # 距 Windows 可用工作区底边的距离
```

这些配置只影响客户端界面，不改变服务端模型或识别协议。界面初始化失败时，客户端会记录警告并继续提供语音输入。

## 识别管线增强

一次语音输入在长按确认时获得唯一任务 ID。准备、录音、识别和隐藏事件使用同一 ID；如果较早任务的结果延迟到达，状态岛会忽略它，避免旧结果覆盖新任务的状态。

正式录音开始时还会登记一个单调递增的本地送达序号。服务端结果即使乱序返回，也会先进入内存队列，严格按录音登记顺序逐段上屏；前序任务失败或取消时会自动跳过，不会永久阻塞后续结果。

## 使用 UV 构建双包

维护者在 Windows PowerShell 中运行：

```powershell
./scripts/build-windows-packages.ps1
```

脚本使用 `uv.lock` 创建可复现环境，并依次生成：

- `VoxCaps`：Client + Server 完整包；
- `VoxCaps-Client`：只含 Client 的局域网客户端包。

ZIP 文件输出到 `release`。本机需要 Python 和 uv；若尚未安装 uv，可运行 `python -m pip install uv`。ZIP 使用 Python 标准库生成，不再依赖 7-Zip。普通用户不需要安装 uv，直接使用发布页中的 ZIP 即可。

构建脚本会依次执行完整单元测试、源码编译、SVG/ICO 图层检查、三个 EXE 图标资源检查，以及两个 ZIP 的内容与共享行为文件一致性检查；任一步失败都不会生成“成功”提示。

## Windows 手工验收清单

1. 启动客户端，确认预热后任务栏麦克风指示消失，静置时不持续占用。
2. 短按 CapsLock，确认大小写正常切换；麦克风指示可以短暂闪现，但状态岛不出现，松开后设备关闭。
3. 长按 CapsLock 超过 250 毫秒，确认依次出现琥珀准备态、青蓝录音态；松开后进入紫色识别态并关闭麦克风。
4. 快速连续录制两段，确认识别结果按录音开始顺序上屏。
5. 确认右 Alt 与 AltGr 保持 Windows 默认行为，不触发录音；鼠标 X2 仍可按原配置触发。
6. 临时关闭“允许桌面应用访问麦克风”或使用测试替身模拟打开失败，确认客户端保持运行，只显示通用权限提示；恢复权限后下一次长按可重试。
7. 检查完整包和纯客户端包的窗口、托盘和 EXE 均使用同一 VoxCaps 图标。
