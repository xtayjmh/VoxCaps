# VoxCaps 领域词汇

## 桌面语音反馈

### 识别态

用户结束录音后、文字结果尚未送达当前输入位置之前的可见反馈状态。

### 送达

识别结果已经成功写入当前输入位置。送达不是独立的可见状态：成功后灵动岛立即隐藏，不显示绿色完成动画。

## 语音触发

### 延迟触发键

CapsLock 是唯一的键盘延迟触发键：250 毫秒内松开属于短按并正常切换大小写，持续按住超过 250 毫秒后开始语音输入，松开结束。

### 麦克风预热

客户端托盘就绪约 1 秒后，在后台对麦克风执行一次可用性验证并立即释放，最长等待 1.5 秒。预热不意味着客户端在后台持续占用麦克风；预热失败也不阻止客户端继续运行，后续语音触发仍会重试。

### 候选录音

延迟触发键刚按下、尚未超过长按阈值期间临时接收的音频。短按时丢弃，确认长按后并入正式录音，用于避免漏掉开头语音。

候选录音只存在于内存中，不保存、不识别、不发送。短按仍会短暂使用麦克风并可能触发 Windows 麦克风指示，但释放按键后必须立即关闭设备并丢弃全部候选音频。

### 键盘语音键

客户端仅支持 CapsLock 作为键盘长按语音键，以避免修饰键兼容问题。鼠标 X2 仍可作为独立触发源共存。

### 连续口述

上一段语音仍在识别时，用户可以立即开始下一段录音。每段录音都是独立语音会话，早先会话的迟到状态不得覆盖当前会话的桌面反馈。

连续口述的识别结果始终按照录音开始顺序送达输入位置。后开始的会话即使先完成识别，也必须等待更早会话完成送达。

### 麦克风不可用

系统麦克风权限、安全软件策略、设备占用或设备异常导致语音输入无法开始。客户端仅通过 Windows 托盘通知提示用户检查麦克风权限或安全软件设置，不点名具体安全产品，也不显示错误灵动岛；下一次语音触发仍会自动重试。

CapsLock 已按住超过 250 毫秒、但麦克风不可用时，该次操作仍视为语音意图，不补发 CapsLock 的系统默认功能。

### 准备态

语音触发已经超过 250 毫秒、但麦克风尚未完成打开时的短暂桌面反馈。准备态使用琥珀色动画，与青蓝色录音态区分；麦克风就绪后转为录音态，等待超过 1.5 秒则隐藏并提示麦克风不可用。

## 品牌视觉

### VoxCaps 标志

VoxCaps 的主标志使用深色圆角方形底与青蓝渐变声波，声波整体形成简化的字母 V。标志不依赖完整文字或细小麦克风图案，以保证 Windows 小尺寸图标仍可辨识。
